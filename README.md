# Электронный дневник (Fullstack, Express + PostgreSQL)

Live demo: https://programma-trofima-dnevnik.vercel.app/

Это прототип электронного дневника с ролью `parent` (родитель) и `teacher` (учитель).
Проект интересен для трудоустройства, потому что включает полноценный fullstack-цикл: авторизация, работа с БД, API, динамический UI и деплой.

## Стек

- Backend: `Express` + `TypeScript`
- Авторизация: `cookie-session`
- База данных: `PostgreSQL` (`pg`)
- Деплой: `Vercel` (server-side function + статические ассеты из `public/`)

## Ключевые фичи

- Роли: `parent` и `teacher`
- Диалог вход/регистрация/выход
- Просмотр дневника (уроки по датам), успеваемость и итоговые
- Редактирование учителем химии: оценки/отсутствие
- Добавление ученика родителем и создание классов учителем
- Демо-режим:
  - если `DATABASE_URL` не задан, часть данных работает из in-memory “заглушек”
  - если `DATABASE_URL` задан, можно заполнить БД и работать с реальными таблицами

## Как запустить локально

### 1) Установить зависимости

```bash
npm install
```

### 2) Создать переменные окружения

Скопируй `.env.example` в `.env` и укажи `DATABASE_URL` и `SESSION_SECRET`.
Секреты держи локально (файл `.env` игнорируется).

Пример:

```env
DATABASE_URL=postgresql://USER:PASSWORD@HOST/neondb?sslmode=require
SESSION_SECRET=...some_random_string...
```

### 3) (Опционально) Заполнить БД тестовыми данными

```bash
npm run db:seed
```

> Примечание: seed создаёт/обновляет схему и наполняет данные для демонстрации.

> Демо-пользователи (с готовыми login/pass) будут добавлены только если в `.env`
> выставлен `SEED_DEMO_USERS=1`.

### 4) Запуск

- Разработка (TypeScript с автоподгрузкой):
  ```bash
  npm run dev
  ```
- Прод (через сборку в `dist/`):
  ```bash
  npm run build
  npm start
  ```

После запуска приложение доступно по:

- `http://127.0.0.1:3000`

## Основные API (backend)

Маршруты находятся в `src/routes/*`:

- `src/routes/auth.ts`:
  - `POST /api/auth/login`
  - `POST /api/auth/register`
  - `POST /api/auth/logout`
  - `GET /api/auth/me`
- `src/routes/profile.ts`:
  - профиль текущего пользователя
  - управление детьми/классами
- `src/routes/api.ts`:
  - данные для родителя: дневник, оценки, встречи
- `src/routes/teacher.ts`:
  - данные для учителя и редактирование уроков химии

Все роуты проксируются под `/api` (см. `app.ts` и `vercel.json`).

## Что показать рекрутеру (быстрый сценарий)

1) Войти как `parent` и выбрать класс/ученика.
2) Открыть “Журнал”, сменить дату и посмотреть уроки.
3) Переключиться на “Успеваемость” и “Итоговые”.
4) Войти как `teacher`:
   - открыть “Журнал”
   - выбрать день
   - открыть “Редактировать урок”
   - изменить оценку/отсутствие для урока химии

## Папки

- `src/` — backend (роуты, авторизация, слой работы с БД)
- `public/` — фронтенд (SPA UI: `index.html`, `css/`, `js/`)
- `api/` — entrypoint для Vercel serverless function

## Примечание по безопасности

Файл `.env` с секретами не должен попадать в репозиторий (`.env` игнорируется).
Локальные демо-доступы, необходимые для проверки функционала, лучше хранить отдельно и/или контролировать через параметры запуска.

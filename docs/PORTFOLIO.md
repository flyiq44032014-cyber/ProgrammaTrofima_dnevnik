# Портфолио: как подать проект (для fullstack)

## Одним предложением

Проект “Электронный дневник” — fullstack приложение с role-based доступом, авторизацией через cookie-сессии и PostgreSQL-данными (Express + TypeScript), с деплоем на Vercel.

## Почему этот проект подходит для работы

- Есть и фронтенд, и backend (не “только верстка” или “только API”)
- Есть безопасность по базовому уровню:
  - хэширование паролей (`bcrypt`)
  - middleware role-check для API
  - параметризация запросов в БД (через `pg`)
- Есть “реальная” задача продуктового уровня:
  - дневник по датам
  - успеваемость и итоговые
  - разный функционал для родителя и учителя

## Готовый шаблон кейса

Используй этот шаблон в описании репозитория/на собеседовании:

1) Проблема: “Нужен удобный дневник для родителей/учителей с быстрым просмотром по датам”.
2) Решение: “Сделал SPA + REST API, где UI переключает состояния и вызывает эндпоинты для данных”.
3) Архитектурные решения:
   - единый backend Express для API и статики
   - типизация результатов и разделение слоёв `routes -> services/repository -> db`
4) Что реализовано (фичи):
   - дневник, успеваемость, итоговые
   - создание детей/классов
   - редактирование химии учителем
5) Деплой/эксплуатация:
   - Vercel, serverless entrypoint
   - конфигурация через `.env`
6) Что улучшил/что планирую:
   - доп. security (CSRF/rate-limit/headers)
   - тесты и мониторинг

## Что показать за 2–3 минуты

1) Как выглядит вход и смена роли (`parent` vs `teacher`).
2) Переход по датам в “Журнал”.
3) Переход к “Успеваемость” (диаграммы/таблицы).
4) Для учителя: “Редактировать урок” (оценка/отсутствие).
